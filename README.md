# .github Repository for SpotBridge Development

This repository contains default community health files and configurations for the SpotBridge Development organization. These files provide consistent templates and standards across all repositories in the organization.

## Repository Contents

### 📋 Organization Profile
- **`profile/README.md`** - Organization welcome page displayed on GitHub

### 🔄 Workflow Templates
- **`workflow-templates/`** - Reusable GitHub Actions workflows
  - `ci.yml` - Standard CI workflow for Node.js projects
  - `deploy.yml` - Deployment workflow template

### 📝 Issue Templates  
- **`ISSUE_TEMPLATE/`** - Standardized issue templates
  - `bug_report.yml` - Bug report template
  - `feature_request.yml` - Feature request template
  - `config.yml` - Issue template configuration

### 🔀 Pull Request Template
- **`PULL_REQUEST_TEMPLATE.md`** - Standard PR template for consistent reviews

### 💰 Funding Configuration
- **`FUNDING.yml`** - Sponsorship and funding links

## Usage

These files automatically apply to all repositories in the SpotBridge Development organization:

- **Issue Templates**: Available when creating new issues in any org repository
- **PR Template**: Auto-populated when creating pull requests  
- **Workflow Templates**: Available in the Actions tab of any org repository
- **Organization Profile**: Displayed on the [SpotBridge Development](https://github.com/spotbridgedev) organization page

## Contributing

To modify these organization-wide defaults:

1. Fork this repository
2. Make your changes
3. Submit a pull request with a clear description
4. Changes will apply to all org repositories once merged
